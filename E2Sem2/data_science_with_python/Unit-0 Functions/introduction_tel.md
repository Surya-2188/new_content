


# చాప్టర్: పైథాన్ ఫంక్షన్స్ (Functions)

**(ప్రోగ్రామింగ్‌లో అత్యంత ఫ్లెక్సిబుల్ కాన్సెప్ట్)**

**లక్ష్యం:** కేవలం కోడ్ రాయడం కాకుండా, తెలివిగా (Smart) కోడ్ రాయడం నేర్చుకోవడం.


-----


## 1️⃣ అసలు పైథాన్ ఫంక్షన్స్ ఎందుకు స్పెషల్?

C లేదా Java లో, ఫంక్షన్స్ చాలా స్ట్రిక్ట్ గా ఉంటాయి:


 * ఆర్డర్ (Order) ఖచ్చితంగా పాటించాలి.
 * డీఫాల్ట్ విలువలు (Default values) ఉండవు.
 * కీవర్డ్ ఆర్గ్యుమెంట్స్ (Keyword arguments) ఉండవు.
 * వేరే రకమైన ఇన్‌పుట్స్ కావాలంటే, ఒకే ఫంక్షన్ ని 5 సార్లు రాయాలి (**Function Overloading**).


**పైథాన్ స్టైల్:**
పైథాన్ ఏమంటుందంటే:


> *"కేవలం చిన్న చిన్న మార్పుల కోసం ఒకే కోడ్ ని 5 సార్లు ఎందుకు రాయాలి?"*


అందుకే పైథాన్ మనకు అద్భుతమైన ఫీచర్స్ ఇచ్చింది:


1. **Positional Arguments** (మామూలు పద్ధతి).
2. **Keyword Arguments** (ఆర్డర్ తో పనిలేదు).
3. **Default Arguments** (ఆప్షనల్ ఇన్‌పుట్స్).
4. **Multiple Returns** (ఒకేసారి చాలా విలువలు వెనక్కి ఇవ్వడం).


దీనివల్ల **ఒకే ఒక్క ఫంక్షన్**, 10 రకాల పనులను చేయగలదు.

## PART A — ఫంక్షన్ ని డిఫైన్ చేయడం

### 1️⃣ సింపుల్ ఫంక్షన్ (No Parameters)

ఇది కాలిక్యులేటర్ లో "Clear Button" లాంటిది. బటన్ నొక్కగానే పని జరుగుతుంది.


**ఉదాహరణ 1: Greeting**


```python
def greet():
   print("హలో! పైథాన్ క్లాస్ కి స్వాగతం.")


# ఫంక్షన్ ని పిలవడం (Call)
greet()
```


**✔ లైన్-బై-లైన్:**


 * `def`: ఫంక్షన్ మొదలుపెట్టడానికి కీవర్డ్.
 * `greet`: ఫంక్షన్ పేరు.
 * `()`: ఖాళీ బ్రాకెట్స్ (ఇన్‌పుట్ లేదు).
 * `:`: ఇక్కడితో హెడ్డింగ్ అయిపోయింది, బాడీ మొదలవుతుంది.
 * `greet()`: ఇది రాస్తేనే కోడ్ రన్ అవుతుంది.


**సాధారణ తప్పులు:**


1. **బ్రాకెట్స్ మర్చిపోవడం:** `greet` అని రాస్తే ఫంక్షన్ రన్ అవ్వదు, `greet()` అని రాయాలి.
2. **Indentation:** ఫంక్షన్ లోపల రాయాల్సిన కోడ్ ని బయట రాయడం.

## PART B — పారామీటర్స్ తో ఫంక్షన్స్

### 2️⃣ Positional Arguments (వరుస క్రమం ముఖ్యం)

ఇది C లేదా Java లాంటిదే. ఆర్డర్ (Order) ముఖ్యం. ఫంక్షన్ (పేరు, వయస్సు) అడిగితే, మీరు (వయస్సు, పేరు) ఇవ్వకూడదు.


**ఉదాహరణ 1: స్టూడెంట్ వివరాలు**


```python
def student(name, roll_no):
   print("పేరు:", name)
   print("రోల్ నం:", roll_no)


# సరైన ఆర్డర్
student("రాజు", 45)
```


**విశ్లేషణ:**


 * `"రాజు"` మొదటి స్థానంలో ఉంది → `name` కి వెళ్తుంది.
 * `45` రెండో స్థానంలో ఉంది → `roll_no` కి వెళ్తుంది.


**తప్పు:** `student(45, "రాజు")` అని రాస్తే, పేరు 45 అని ప్రింట్ అవుతుంది. ఇది లాజికల్ గా తప్పు.


-----


###  TYPE 3 — Keyword Arguments (పైథాన్ స్పెషల్)
**తప్పు:** `student(45, "రాజు")` అని రాస్తే, పేరు 45 అని ప్రింట్ అవుతుంది. ఇది లాజికల్ గా తప్పు.


-----


### 3️⃣ Keyword Arguments (పైథాన్ స్పెషల్)

స్విగ్గీలో ఫుడ్ ఆర్డర్ చేయడం లాంటిది.
```python
def order(food, drink):
   print(f"ఆర్డర్: {food} మరియు {drink}")


# ఇక్కడ ఆర్డర్ ముఖ్యం కాదు!
order(drink="లైమ్ సోడా", food="బిర్యానీ")
```


**✔ లాభం:** పేర్లు వాడుతున్నాం కాబట్టి, ఏది ఎక్కడికి వెళ్తుందో క్లియర్ గా తెలుస్తుంది.

## PART C — DEFAULT ARGUMENTS

### 4️⃣ Default Parameters

రస్టారెంట్ లో "భోజనం థాలి" లాంటిది. అందులో డీఫాల్ట్ గా పచ్చడి వస్తుంది. మీరు ప్రత్యేకంగా కోరితేనే మారుస్తారు.


**ఉదాహరణ 1: డీఫాల్ట్ గ్రీటింగ్**


```python
def greet(name="Student"):
   print("హలో,", name)


greet()         # పేరు ఇవ్వలేదు -> "హలో, Student" వస్తుంది.
greet("ఉదయ్")   # పేరు ఇచ్చాం -> "హలో, ఉదయ్" వస్తుంది.
```


**ఉదాహరణ 2: పవర్ (Power) కాలిక్యులేషన్**


```python
def power(base, exp=2):  # స్క్వేర్ డీఫాల్ట్
   return base ** exp


print(power(5))      # 5^2 = 25 (డీఫాల్ట్ వాడింది)
print(power(5, 3))   # 5^3 = 125 (మనం ఇచ్చింది వాడింది)
```


**చేసే తప్పులు:**


1. **ఆర్డర్ తప్పు:** డీఫాల్ట్ ఆర్గ్యుమెంట్స్ ఎప్పుడూ చివర్లో ఉండాలి. `def wrong(a=5, b):` అని రాయకూడదు.
2. **Mutable Defaults (ఇంటర్వ్యూ ప్రశ్న):** `list=[]` ని డీఫాల్ట్ గా వాడకండి. అది పాత డేటాని గుర్తుపెట్టుకుంటుంది. `None` వాడటం మంచిది.

## PART D — RETURN VALUES

### 5️⃣ Return Statements

`print` అంటే ATM స్క్రీన్ మీద బ్యాలెన్స్ చూడటం లాంటిది. `return` అంటే ATM నుంచి డబ్బులు బయటకు రావడం లాంటిది.


**ఉదాహరణ:**


```python
def sum(a, b):
   return a + b  # విలువను వెనక్కి ఇస్తుంది


result = sum(3, 7)  # వచ్చిన 10 ని 'result' లో స్టోర్ చేసుకున్నాం
print(result)
```


**తప్పు:** `return` రాయకపోతే, ఆ ఫంక్షన్ చేసిన పని వృధా అవుతుంది (స్టోర్ చేసుకోలేము).


-----


**తప్పు:** `return` రాయకపోతే, ఆ ఫంక్షన్ చేసిన పని వృధా అవుతుంది (స్టోర్ చేసుకోలేము).

## PART E — NONE (ఏమీ లేదు)

### 6️⃣ Functions Returning `None`

ఒకవేళ ఫంక్షన్ ఏమీ `return` చేయకపోతే, పైథాన్ ఆటోమేటిక్ గా "ఖాళీ బాక్స్" (None) ని ఇస్తుంది.


**ఉదాహరణ:**
```
x = show()
print(x)   # అవుట్‌పుట్: None
```

## PART F — అన్నింటినీ కలపడం (MIXING)

### 7️⃣ రూల్స్

మనం Positional, Keyword, Default అన్నింటినీ ఒకేసారి వాడొచ్చు. కానీ ఒక రూల్ ఉంది: ముందు అన్నం (Positional), తర్వాతే కూరలు (Keyword/Default).


**ఉదాహరణ: బిల్లింగ్** (Keyword), tip (Default)
print(bill(100, tax=10))



**తప్పు:** `bill(tax=10, 100)` -\> ఎర్రర్\! కీవర్డ్ తర్వాత పొజిషనల్ రాకూడదు.
