

## 1\. జావా ప్రోగ్రామ్ అంటే ఏమిటి?


జావా ప్రోగ్రామ్ అంటే కంప్యూటర్ చేత ఒక పని చేయించడానికి మనం ఇచ్చే ఇన్స్ట్రక్షన్స్ (Instructions).


 * మనం ఇంగ్లీష్ లాంటి పదాలతో కోడ్ రాస్తాం.
 * దాన్ని `.java` ఫైల్ గా సేవ్ చేస్తాం.
   కానీ కంప్యూటర్ కి ఇంగ్లీష్ అర్థం కాదు, దానికి కావాల్సింది మెషీన్ కోడ్.

## 2\. బైట్‌కోడ్ (Bytecode) ఎందుకు కావాలి?


C లాంగ్వేజ్ లో కంపైలర్ డైరెక్ట్ గా మెషీన్ కోడ్ ఇస్తుంది. కానీ జావాలో మధ్యలో **బైట్‌కోడ్** అనే స్టెప్ ఉంటుంది.


**ప్రాసెస్:**


1. మీరు రాసిన కోడ్ (`.java`).
2. **కంపైలర్ (`javac`)**: దీన్ని చెక్ చేసి **Bytecode** (`.class` ఫైల్) గా మారుస్తుంది.
3. **JVM**: ఈ బైట్‌కోడ్ ని తీసుకుని మీ కంప్యూటర్ కి అర్థమయ్యేలా మారుస్తుంది.


**ఉపయోగం:**


 * **Platform Independence:** మీరు Windows లో రాసిన కోడ్ ని, మార్చకుండానే Mac లేదా Linux లో రన్ చేయొచ్చు. ("ఒకచోట రాయి, ఎక్కడైనా రన్ చేయి").

## 3\. క్లాస్ (Class) మరియు ఆబ్జెక్ట్ (Object) - (చాలా ముఖ్యం\!)


### A) C కి జావా కి తేడా


C లాంగ్వేజ్ లో `printf` లాంటి ఫంక్షన్స్ బయట స్వేచ్ఛగా ఉంటాయి.
కానీ జావాలో రూల్ స్ట్రిక్ట్: **"ఏ ఫంక్షన్ (Method) అయినా, ఏ వేరియబుల్ అయినా క్లాస్ లోపలే ఉండాలి."**


### B) క్లాస్ అంటే బ్లూప్రింట్


క్లాస్ అనేది ఒక బాక్స్ లాంటిది. అందులో డేటా (Variables) మరియు పనులు (Methods) ఉంటాయి.
ఉదాహరణకు: `System.out.println` తీసుకుంటే:


 * `System` అనేది క్లాస్.
 * `println` అనేది మెథడ్.
   మెథడ్స్ ఎప్పుడూ క్లాస్ లేదా ఆబ్జెక్ట్ లోపలే ఉంటాయి.


### C) "Int vs Variable" అనాలజీ (సింపుల్ గా అర్థం చేసుకోవడానికి)


ఆబ్జెక్ట్ అంటే ఏంటో తెలియక చాలామంది కన్ఫ్యూజ్ అవుతారు. ఇది చూడండి:


మనం `int` (డేటా టైప్) ని డైరెక్ట్ గా వాడలేం కదా? ఒక వేరియబుల్ క్రియేట్ చేయాలి.


```c
int a;    // 'int' అనేది టైప్, 'a' అనేది వేరియబుల్
```


అలాగే జావాలో, **Class** అనేది కూడా ఒక టైప్ మాత్రమే. దాన్ని వాడాలంటే మనం ఒక వేరియబుల్ క్రియేట్ చేయాలి. ఆ వేరియబుల్ పేరే **Object**.


```java
Car myCar;   // 'Car' అనేది క్లాస్ (టైప్), 'myCar' అనేది ఆబ్జెక్ట్ (వేరియబుల్)
```


**తేడా చూడండి:**


 * `int` $\rightarrow$ డేటా టైప్ అయితే, `a` వేరియబుల్.
 * `Car` $\rightarrow$ క్లాస్ (టైప్) అయితే, `myCar` ఆబ్జెక్ట్.


> **గుర్తుంచుకోండి:** ఆబ్జెక్ట్ అంటే క్లాస్ అనే టైప్ కి సంబంధించిన ఒక వేరియబుల్ మాత్రమే.

## 4\. `public static void main` అర్థం ఏంటి?


ప్రోగ్రామ్ స్టార్ట్ అయినప్పుడు మన దగ్గర ఇంకా ఆబ్జెక్ట్స్ ఏమీ ఉండవు. మరి ప్రోగ్రామ్ ఎలా స్టార్ట్ అవ్వాలి? అందుకే `main` ని ఇలా రాస్తాం:


1. **`public`**: ఇది అందరికీ అందుబాటులో ఉంటుంది (JVM దీన్ని కాల్ చేయాలి కాబట్టి).
2. **`static`**: (ముఖ్యం) మామూలుగా మెథడ్ ని పిలవాలంటే ఆబ్జెక్ట్ కావాలి. కానీ `static` అని రాస్తే, **ఆబ్జెక్ట్ లేకుండానే** కాల్ చేయొచ్చు. ప్రోగ్రామ్ స్టార్టింగ్ లో ఆబ్జెక్ట్ ఉండదు కాబట్టి ఇది స్టాటిక్ అయ్యి ఉండాలి.
3. **`void`**: ఈ మెథడ్ రన్ అయ్యి రిజల్ట్ ని వెనక్కి ఇవ్వదు.
4. **`main`**: ప్రోగ్రామ్ ఇక్కడి నుండే మొదలవ్వాలని JVM కి తెలుసు.

## 5\. హలో వరల్డ్ ప్రోగ్రామ్ (Hello World)


```java
public class HelloWorld {
   public static void main(String[] args) {
       System.out.println("Hello World");
   }
}
```


**వివరణ:**


 * `public class HelloWorld`: మనం `HelloWorld` అనే క్లాస్ (బ్లూప్రింట్) ని క్రియేట్ చేసాం.
 * `public static void main`: JVM ఇక్కడికి వచ్చి ప్రోగ్రామ్ ని స్టార్ట్ చేస్తుంది. ఆబ్జెక్ట్ లేకపోయినా `static` వల్ల ఇది రన్ అవుతుంది.
 * `System.out.println(...)`: ఇది స్క్రీన్ మీద టెక్స్ట్ ని ప్రింట్ చేస్తుంది.
