

#  లెక్చర్: ఆబ్జెక్ట్ ఓరియెంటెడ్ ప్రోగ్రామింగ్ (Student Example)


**ముఖ్య ఉద్దేశ్యం:** స్టూడెంట్ మేనేజ్‌మెంట్ సిస్టమ్ ద్వారా OOP సింటాక్స్, మెమరీ లాజిక్ మరియు ఆబ్జెక్ట్స్ ఎలా వేరు వేరుగా పని చేస్తాయో నేర్చుకోవడం.


-----


## 1️⃣ అసలు లక్ష్యం: స్టూడెంట్ మేనేజ్‌మెంట్ సిస్టమ్ తయారు చేయడం


మనం ఇప్పుడు ఒక కాలేజీ సాఫ్ట్‌వేర్ తయారు చేస్తున్నాం అనుకోండి. మనకు కావాల్సింది ఇది:


1. **ఒక టెంప్లేట్ (Class):** అసలు "స్టూడెంట్" అంటే ఎవరు? వాళ్లకి ఏమేమి ఉంటాయి? (పేరు, రోల్ నంబర్, బ్రాంచ్) అనేది డిజైన్ చేయాలి.
2. **నిజమైన స్టూడెంట్స్ (Objects):** ఆ టెంప్లేట్ వాడి మనం నిజమైన స్టూడెంట్స్ ని సృష్టించాలి. (ఉదాహరణకు: రాజు, రాణి).
3. **ప్రూఫ్ (Proof):** రాజు డేటా మారుస్తే, రాణి డేటా మారకూడదు. ఎవరి డేటా వాళ్ళదే ఉండాలి.


-----


## 2️⃣ కోడ్: క్లాస్ ని స్టెప్-বাই-స్టెప్ గా రాద్దాం


### Step A: బ్లూప్రింట్ (Class) ని డిజైన్ చేయడం


ముందు క్లాస్ రాద్దాం. గుర్తుంచుకోండి, ఇది కేవలం డిజైన్ మాత్రమే. మెమరీలో ఇంకా స్థలం తీసుకోదు.


```java
class Student {
   // 1. ATTRIBUTES (డేటా - ఖాళీ బాక్సులు)
   String name;
   int roll;
   String branch;


   // 2. BEHAVIOR (పనితనం - మెథడ్స్)
   void study() {
       System.out.println(name + " is studying in the " + branch + " lab.");
   }


   void showIdentity() {
       System.out.println("ID: " + roll + " | Name: " + name + " | Branch: " + branch);
   }
}
```


### Step B: అమలు చేయడం (Objects సృష్టించడం)


ఇప్పుడు `main` method లోపల, ఆ బ్లూప్రింట్ కి ప్రాణం పోద్దాం.


```java
public class CollegeApp {
   public static void main(String[] args) {
      
       // 1. మొదటి స్టూడెంట్ (రాజు) ని సృష్టించడం
       Student s1 = new Student();
       s1.name = "Raju";
       s1.roll = 101;
       s1.branch = "CSE";


       // 2. రెండో స్టూడెంట్ (రాణి) ని సృష్టించడం
       Student s2 = new Student();
       s2.name = "Rani";
       s2.roll = 102;
       s2.branch = "ECE";


       // 3. వాళ్ల ఐడెంటిటీ చూపించడం
       s1.showIdentity();
       s2.showIdentity();
   }
}
```

## 3️⃣ అసలైన లాజిక్: మెమరీ ఐసోలేషన్ (Memory Isolation)


*(ఇది ఈ లెక్చర్‌లో అత్యంత ముఖ్యమైన భాగం. దీన్ని విద్యార్థులకు నిదానంగా వివరించండి.)*


### "అప్లికేషన్ ఫారమ్" ఉదాహరణ


కాలేజీ ఆఫీసులో ఒక కట్ట **ఖాళీ అప్లికేషన్ ఫారమ్‌లు** ఉన్నాయి అనుకోండి.


 * **Class (`Student`)** అనేది ఆ **ఖాళీ ఫారమ్** లాంటిది.
 * **Object (`s1`, `s2`)** అనేది నింపిన **నిజమైన ఫారమ్**.


రాజు తన ఫారమ్ నింపి "CSE" అని రాశాడు.
రాణి తన ఫారమ్ నింపి "ECE" అని రాసింది.


**ముఖ్యమైన పాయింట్:** ఇప్పుడు రాజు రబ్బర్ తీసుకుని తన ఫారమ్ మీద బ్రాంచ్ తుడిపేసి "Mech" అని రాస్తే, **రాణి పేపర్ మీద మారుతుందా? మారదు కదా\!** ఆమె పేపర్ వేరు, ఇతని పేపర్ వేరు.


### జావా (Java) లో ఇది ఎలా జరుగుతుంది?


1. **`new` అనే పదం (Keyword):**
   మీరు ఎప్పుడైతే `new Student()` అని రాస్తారో, జావా **Heap Memory** లోకి వెళ్లి ఒక **కొత్త బాక్స్ (Brand New Box)** ని తయారు చేస్తుంది.
2. **వేరు వేరు వేరియబుల్స్:**
     * `s1` కి తన సొంత `name`, `roll` బాక్సులు ఉంటాయి.
     * `s2` కి తన సొంత `name`, `roll` బాక్సులు ఉంటాయి.
3. **లాజిక్ ఒకటే:**
   ఇద్దరూ `showIdentity()` అనే ఒకే కోడ్ ని వాడుతారు, కానీ ఆ కోడ్ రన్ అయ్యేటప్పుడు ఎవరి బాక్స్ లో డేటా వాళ్ళు వాడుకుంటారు.

## 4️⃣ ప్రూఫ్ (Proof): ఒక ఆబ్జెక్ట్ ని మారిస్తే ఏమవుతుంది?


`s1` ని మారిస్తే `s2` కి ఏమీ కాదని కోడ్ ద్వారా నిరూపిద్దాం.


```java
public class CollegeApp {
   public static void main(String[] args) {
      
       // SETUP: ఇద్దరు స్టూడెంట్స్ ని క్రియేట్ చేద్దాం
       Student s1 = new Student();
       s1.name = "Raju";
       s1.roll = 101;


       Student s2 = new Student();
       s2.name = "Rani";
       s2.roll = 102;


       // TEST 1: మార్చకముందు ప్రింట్ చేద్దాం
       System.out.println("--- Before Change ---");
       s1.showIdentity(); // రాజు అని వస్తుంది
       s2.showIdentity(); // రాణి అని వస్తుంది


       // TEST 2: కేవలం రాజుని మాత్రమే మారుద్దాం
       System.out.println("--- Modifying Raju's Roll Number ---");
       s1.roll = 999;


       // TEST 3: మళ్ళీ ఇద్దరినీ చెక్ చేద్దాం
       System.out.println("--- After Change ---");
       System.out.println("s1 Roll: " + s1.roll); // ఇది 999 కి మారింది
       System.out.println("s2 Roll: " + s2.roll); // ఇది 102 గానే ఉంది! (Safe)
   }
}
```


**ముగింపు:** `s1` ని మారిస్తే `s2` మీద **ఎలాంటి ప్రభావం లేదు**. దీన్నే "మెమరీ ఐసోలేషన్" అంటారు.

## 5️⃣ మెమరీలో ఇది ఎలా కనిపిస్తుంది? (Visualization)


*(ఈ డయాగ్రామ్ ని బోర్డు మీద గీయండి)*


కోడ్ రన్ అయినప్పుడు, మెమరీ ఇలా ఉంటుంది:


**HEAP MEMORY**


```text
-----------------------------       -----------------------------
|  Address: 500 (s1)        |       |  Address: 800 (s2)        |
|                           |       |                           |
|  name   : "Raju"          |       |  name   : "Rani"          |
|  roll   : 999             |       |  roll   : 102             |
|  branch : "CSE"           |       |  branch : "ECE"           |
-----------------------------       -----------------------------
```


 * `s1` అనేది బాక్స్ నంబర్ 500 ని చూపిస్తోంది.
 * `s2` అనేది బాక్స్ నంబర్ 800 ని చూపిస్తోంది.
 * రెండూ భౌతికంగా వేరు వేరు చోట్ల ఉన్నాయి.

## 6️⃣ విద్యార్థులు చేసే సాధారణ తప్పులు (Common Mistakes)


*(ల్యాబ్ లో ఎర్రర్స్ రాకుండా ఉండాలంటే ఇది తప్పక చదవాలి)*


### ❌ 1. దెయ్యం ఆబ్జెక్ట్ (NullPointerException)


**తప్పు:** వేరియబుల్ క్రియేట్ చేసి, ఆబ్జెక్ట్ క్రియేట్ చేయకపోవడం.


```java
Student s1;
s1.name = "Raju"; // ప్రోగ్రామ్ క్రాష్ అవుతుంది!
```


**ఎందుకు:** మీరు `s1` అనే లేబుల్ అతికించారు కానీ, డబ్బా (`new Student()`) కొనలేదు. `s1` దేన్నీ చూపించట్లేదు (Null).
**సొల్యూషన్:** ఎప్పుడూ `new` వాడాలి: `Student s1 = new Student();`


### ❌ 2. విగ్రహం తప్పు (Static Error)


**తప్పు:** ఆబ్జెక్ట్ పేరు చెప్పకుండా మెథడ్ ని పిలవడం.


```java
public static void main(String[] args) {
   showIdentity(); // ఎర్రర్ వస్తుంది
}
```


**ఎందుకు:** మీరు గాలిలో "ఐడెంటిటీ చూపించు\!" అని అరుస్తున్నారు. జావాకి అర్థం కాదు, ఎవరిని అడుగుతున్నారో. రాజునా? రాణినా?
**సొల్యూషన్:** కచ్చితంగా ఆబ్జెక్ట్ పేరు చెప్పాలి: `s1.showIdentity();`


### ❌ 3. స్కోప్ మతిమరుపు (Scope Shadowing)


**తప్పు:** మెథడ్ లోపల మళ్ళీ వేరియబుల్స్ ని డిక్లేర్ చేయడం.


```java
void setRoll(int r) {
   int roll = r; // లాజిక్ తప్పు!
}
```


**ఎందుకు:** మెథడ్ లోపల `int roll` అని రాస్తే, అది ఒక **టెంపరరీ వేరియబుల్** అవుతుంది. మీరు ఆ టెంపరరీ దాన్ని మారుస్తున్నారు తప్ప, అసలైన స్టూడెంట్ రోల్ నంబర్ ని కాదు.
**సొల్యూషన్:** లోపల `int` అని రాయకండి. `roll = r;` లేదా `this.roll = r;` అని వాడండి.


### ❌ 4. ఫైల్ పేరు తప్పుగా ఇవ్వడం


**తప్పు:** `main` method `CollegeApp` లో ఉంటే, ఫైల్ ని `Student.java` అని సేవ్ చేయడం.
**సొల్యూషన్:** `main` ఎక్కడ ఉంటే ఆ క్లాస్ పేరే ఫైల్ కి ఉండాలి. `CollegeApp.java` అని సేవ్ చేయండి.


### ❌ 5. మెయిన్ (Main) ని క్లాస్ లోపల రాయడం


**తప్పు:** `main` మెథడ్ ని తీసుకెళ్లి `Student` క్లాస్ లోపల రాయడం.
**ఎందుకు:** ఇది టెక్నికల్ గా పనిచేసినా, పద్ధతి కాదు. బ్లూప్రింట్ (Student) వేరు, ఇల్లు కట్టేవాడు (CollegeApp) వేరు.
**సొల్యూషన్:** రెండింటినీ వేరు వేరు క్లాసుల్లో ఉంచండి.

## 7️⃣ చివరిగా (Summary)


1. **Class:** ఒక టెంప్లేట్ (ఖాళీ అప్లికేషన్ ఫారమ్).
2. **Object:** ఆ టెంప్లేట్ నుంచి తయారైన నిజమైన వస్తువు (నింపిన ఫారమ్).
3. **`new`:** కొత్త మెమరీ బాక్స్ ని సృష్టించే కీవర్డ్.
4. **Isolation:** `s1` మరియు `s2` కోడ్ ని షేర్ చేసుకుంటాయి, కానీ **డేటాని ఎప్పుడూ షేర్ చేసుకోవు**.
5. **Syntax:** ఎప్పుడూ ఇలాగే మొదలుపెట్టాలి -\> `ClassName obj = new ClassName();`.
